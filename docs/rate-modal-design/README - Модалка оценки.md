# Handoff: Модалка оценки игры (Яндекс Игры) — «Изумрудный меридиан»

## Overview
Внутриигровая **pre-prompt модалка**, которая мягко предлагает игроку оценить игру на платформе Яндекс Игры. Появляется на карте кампании после завершения третьего уровня, после короткой спокойной паузы.

**КРИТИЧЕСКИ ВАЖНО:** это только pre-prompt. Модалка **не собирает оценку сама**, **не показывает интерактивные звёзды**, **не имитирует системное окно Яндекс Игр**. По нажатию основной кнопки разработчик открывает официальное нативное rate-окно через SDK платформы (`ysdk.feedback.canReview()` → `ysdk.feedback.requestReview()`). Эта модалка — только вежливая подводка к этому вызову.

Эстетика наследуется от экрана выбора кампаний: экспедиционный архив, латунь, пергамент, восковая печать, компас. Окно выглядит как небольшая карточка из полевого журнала / найденный экспедиционный документ поверх карты.

## About the Design Files
Файл `Модалка оценки.dc.html` — **дизайн-референс в HTML** («Design Component», нужен рантайм `support.js` из этого же пакета). Откройте в браузере — увидите 6 фреймов на паннируемом холсте. Это прототип задуманного вида и поведения, **а не production-код для копирования**. Задача — воссоздать макет средствами вашей кодовой базы (React / Vue / native / Unity UI), используя принятые паттерны. Источник истины по точным значениям — inline-стили в HTML.

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы, состояния. Воссоздавать пиксель-в-пиксель средствами проекта.

## Interactions & Behavior
- **Момент показа:** на карте кампании после завершения 3-го уровня → игрок увидел результат, закрыл его, вернулся на карту → короткая спокойная пауза → fade+scale-in модалки. **Не** показывать поверх уровня, поверх окна результата, во время/сразу после рекламы, одновременно с paywall. Показ — один раз (далее по политике частоты платформы).
- **«Оценить игру»** → кнопка переходит в `loading` (spinner + «Открываем…», все элементы окна disabled) → разработчик вызывает официальное rate-окно Яндекс SDK. Loading кратковременный, не удерживает игрока надолго.
- **«Позже»** → закрыть без оценки, без повторного давления.
- **Крестик / Esc** → закрыть без оценки. Клик по скриму — на усмотрение (рекомендовано: закрывает, эквивалент «Позже»).
- Анимации спокойные, без «казино».

## UX-ограничения (ОБЯЗАТЕЛЬНО соблюсти)
Нельзя: выбираемые звёзды внутри окна · заранее подсвеченные 5 звёзд · текст «Поставьте 5 звёзд» / «Оцените, если нравится» · обещание валюты/подсказки/жизни за оценку · блокировка продолжения игры · скрытая кнопка «Позже» · guilt-текст · вид обязательного системного сообщения · имитация нативного окна Яндекс Игр.

## State Management
- `open: boolean` — видимость; триггерится после 3-го уровня + пауза.
- `loading: boolean` — true с момента клика «Оценить игру» до открытия SDK-окна; блокирует повторный клик и все действия окна.
- `dismissed` / частота показа — хранить в профиле/локально, чтобы не показывать повторно вне политики.
- Перед показом проверять `ysdk.feedback.canReview()` (если платформа недоступна/уже оценено — не показывать модалку вовсе).
- Кнопка «Оценить игру» → `requestReview()`; результат (`{ feedbackSent }`) можно залогировать, но НЕ менять геймплей в зависимости от него.

## Адаптив и переводы
- ≥ 600px — кнопки в строке, CTA `flex:1.45`, secondary `flex:1`.
- < 600px — кнопки вертикально, CTA сверху, на всю ширину.
- Длинный перевод — высота окна растёт, текст переносится (`text-wrap:pretty`), окно не скроллится; при нехватке ширины кнопки переходят в столбец.
- RTL — `dir="rtl"`, крестик слева, иконка-стрелка `scaleX(-1)`, компас и цифры не зеркалятся.
- Работает при увеличенном системном масштабе текста (не фиксировать высоты текстовых блоков жёстко).

## Accessibility
- `role="dialog"`, `aria-modal="true"`, заголовок связан через `aria-labelledby`; focus-trap внутри окна, фокус возвращается на триггер при закрытии.
- Видимый контрастный focus-ring (см. токен); порядок фокуса: CTA → «Позже» → крестик.
- Esc закрывает. Крестик и «Позже» всегда видимы, hit ≥ 44×44.
- Контраст текста ≥ 4.5:1; смысл не передаётся только цветом (иконки + текст).

## Assets
Изображений нет — всё нарисовано кодом:
- **Медальон-компас** — `radial-gradient` + инлайновый SVG-компас. Можно заменить растровой печатью/иконкой из ассетов проекта (сохранить латунную палитру).
- **Иконки** — инлайновые SVG (компас, external-link, крестик, spinner, штамп-линии). Заменить на иконсет проекта, штрих ~1.8–2.4px, цвета токенов.
- **Грейн** — генеративный `feTurbulence` overlay; можно оставить или заменить текстурой бумаги.
- **Бэкдроп** — упрощённый дим/блюр карты кампаний; в реале это живой экран карты под скримом.
