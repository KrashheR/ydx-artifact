# Handoff: Landscape-ревизия экранов · «Изумрудный меридиан»

## Overview
Это пакет передачи в разработку для **доски ревизии горизонтальных (landscape) экранов** мобильной игры «Найди отличия» во вселенной «Изумрудный меридиан» (экспедиционная, винтажно-картографическая эстетика).

Главный экран пакета — `Ревизия Landscape.dc.html` — это обзорная доска, на которой в едином масштабе собраны **восемь мобильных экранов в ориентации 896 × 414** для проверки вмещения контента. Каждое окно доски показывает **живой экран из соответствующего исходного файла** (не скриншот), скадрированный до нужного кадра.

## About the Design Files
Файлы в этом пакете — это **дизайн-референсы, сделанные в HTML** (прототипы, показывающие задуманный вид и поведение), а **не продакшн-код для прямого копирования**. Задача — **воссоздать эти экраны в среде целевого кодовой базы** (React / Vue / SwiftUI / нативно и т.д.), используя её устоявшиеся паттерны, компоненты и библиотеки. Если среды ещё нет — выбрать наиболее подходящий фреймворк для проекта и реализовать дизайн в нём.

Технически прототипы написаны как «Design Components» (`*.dc.html`) с рантаймом `support.js`: разметка инлайновыми стилями + класс логики на ванильном JS. **Не переносите этот рантайм в продакшн** — он нужен только для просмотра прототипов. Откройте любой `.dc.html` в браузере, чтобы увидеть экран.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, радиусы, тени и интерактив. UI следует воссоздавать пиксель-в-пиксель средствами дизайн-системы целевого приложения. Изображения сцен — плейсхолдеры (диагональная штриховка + подпись-моноширинный код), на их место встают реальные иллюстрации игры.

## Screens / Views

Доска ревизии — это контейнер. Реальные продуктовые экраны — это восемь landscape-кадров (896 × 414), перечисленных ниже. В скобках — исходный файл и где искать кадр.

### 0. Доска ревизии — `Ревизия Landscape.dc.html`
- **Назначение:** внутренний обзорный артефакт для дизайн-ревью; в продакшн не идёт.
- **Layout:** тёмная страница (ширина 1980px, паддинг 56/52/76), шапка (надзаголовок + H1 + описание + легенда статусов справа), затем CSS-grid `grid-template-columns: 896px 896px; gap: 52px 48px; justify-content:center`. Каждая ячейка: строка-заголовок (бейдж № + надзаголовок + название + чип статуса) → «окно» 896×414 (`.win`) → строка-заметка ревизии.
- **`.win`:** `width:896;height:414;border-radius:34;overflow:hidden;position:relative;background:#0c100e;border:1px solid rgba(213,195,154,.12);box-shadow:0 28px 64px rgba(0,0,0,.55)`. Внутри — импорт целого экрана-источника, сдвинутый отрицательным `left/top` так, чтобы нужный кадр встал в (0,0) и обрезался окном.

### 7. Игровой экран · Вариант 1 — два снимка рядом (`Игровой экран.dc.html` → «Mobile landscape · Игровой экран»)
- **Назначение:** геймплей «Найди отличия» — сравнение двух картинок.
- **Layout:** одно-рядный HUD 50px (назад | название уровня | таймер `02:14` | счётчик отличий `3/7` | «Подсказка ×2» | пауза) → игровая зона `display:flex;gap:14px`: **снимок A** (оригинал) и **снимок B** (копия) бок о бок, между ними вертикальный разделитель с иконкой. На снимках: найденные отличия (золотые кружки-галочки), промах (красный X на B), зона подсказки (пунктирный пульсирующий круг + ripple).

### 8. Игровой экран · Вариант 2 — флип-карта (`Игровой экран.dc.html` → «Mobile landscape · Игра (флип-карта)»)
- **Назначение:** альтернативный геймплей — **один снимок на весь экран**, переключение между оригиналом (A) и копией (B) переворотом карты.
- **Layout:** тот же HUD 50px → игровая зона `perspective:1900px`: **флип-карта во весь экран** (`transform-style:preserve-3d;transition:transform .62s cubic-bezier(.4,.05,.2,1)`), две грани с `backface-visibility:hidden` — лицо A (front) и оборот B (`transform:rotateY(180deg)`). Сверху по центру — **сегментный индикатор A | B** (активная грань подсвечена золотом). Снизу по центру — **плавающая кнопка-переключатель** (золотая, с иконкой двусторонних стрелок, лёгкий «дышащий» glow `hintsweep`), вне 3D-контекста, чтобы не переворачиваться.
- **Метки:** на каждой грани — найденные отличия (золотые кружки-галочки), на грани B дополнительно зона подсказки (пунктирный пульсирующий круг).

## Interactions & Behavior
- **Флип-карта (экран 8):** клик по плавающей кнопке переключает `flipped` (false⇄true). Карта анимированно поворачивается на 180° по Y (`.62s`, `cubic-bezier(.4,.05,.2,1)`). Меняются: подпись кнопки («Сравнить с копией B» ⇄ «Показать оригинал A») и активный сегмент индикатора A|B. Это единственная стейтовая интеракция; реализуйте как локальный стейт компонента.
- **Скролл-ленты:** Журнал (экран 1) и Главное меню (экран 3) — горизонтальный скролл ряда карточек; Настройки (экран 4) — вертикальный скролл сетки языков.
- **Кастомный скроллбар (тема игры):** применяется к скроллируемым зонам (класс `.scroll`):
  - WebKit: `::-webkit-scrollbar{width:8px;height:8px}`, трек `rgba(213,195,154,.05)` радиус 4, бегунок `linear-gradient(180deg,#d8af63,#a9762f)` радиус 4 + бордер `rgba(21,27,24,.55)`, hover `linear-gradient(180deg,#e7c074,#b3812f)`.
  - Firefox: `scrollbar-width:thin; scrollbar-color:#a9762f rgba(213,195,154,.06)`.
- **Анимации (keyframes):** `pulse` (зоны подсказок), `ripple` (клик по подсказке), `ringpulse` (кольцо текущего узла карты), `drift` (лёгкий туман на карте), `pulseDot`/`hintsweep` (glow). Длительности: подсказка 2.4s, ripple 1.8s, glow кнопки 3.2s.
- **Все интерактивные элементы ≥ 36–44px** по высоте (тач-таргеты).

## State Management
Минимальный стейт, по экранам:
- **Флип-карта (8):** `flipped: boolean`. Триггер — клик кнопки. Производные: текст кнопки, активный сегмент A|B, transform карты.
- **Журнал/Карта:** `completed: number` (0–13) — сколько уровней пройдено; определяет статус каждого уровня/узла (пройден/текущий/закрыт) и заполнение прогресс-сегментов. Текущий = индекс `completed`.
- **Настройки:** `lang: string` (код языка, напр. `ru`) — выбранный язык; выделяет элемент в сетке.
- **Геймплей (7/8):** в проде — таймер обратного отсчёта, набор отличий с координатами (`found`/`remaining`), счётчик промахов и точность. В прототипе значения статичны (таймер `02:14`, найдено `3/7`).

### Типографика
- **Cormorant Garamond** (serif, 500–700) — заголовки, названия уровней/кампаний. Размеры: H1 32–46px, H2 26–42px, названия карточек 15–21px.
- **Manrope** (400–800) — UI-текст, кнопки, подписи. Надзаголовки: 8.5–12px, `letter-spacing .18–.34em`, uppercase, цвет латунь.
- **JetBrains Mono** (400–600) — номера, коды (`REF-0481-N`), таймер, числовые бейджи.
- **Noto Sans Arabic** — для RTL-экранов (присутствуют в источниках, на доске не показаны).

### Радиусы
Рамка телефона landscape `34px`; модалки `16–20px`; карточки `11–14px`; кнопки `7–12px`; чипы/бейджи `5–9px`; превью-тумбы `8px`; прогресс-сегменты `2px`.

### Тени
- Окно/телефон: `0 28–30px 64–70px rgba(0,0,0,.5–.55)`.
- Модалки: `0 40–50px 100–120px rgba(0,0,0,.7), inset 0 1px 0 rgba(213,195,154,.08)`.
- Карточки: `0 12–22px 26–50px rgba(0,0,0,.3–.5)`; активная карточка +`0 0 26px rgba(184,138,69,.14)`.
- Золотая кнопка: `0 8–12px 18–28px rgba(184,138,69,.28–.4), inset 0 1px 0 rgba(255,255,255,.3)`.
- Свечение акцентов: `0 0 8–14px rgba(216,175,99,.5)`.

### Фактуры
- **Зерно (grain):** инлайновый SVG `feTurbulence` (`baseFrequency .85`), `opacity .04–.05; mix-blend-mode:overlay` — лёгкий шум поверх фонов.
- **Плейсхолдер снимка:** `repeating-linear-gradient(45deg,...)` диагональная штриховка + иконка фото + моноширинная подпись.

## Assets
Собственных растровых ассетов нет. Используются: инлайновые SVG-иконки (lucide-подобные), SVG-плейсхолдеры изображений, SVG-шум. Шрифты — Google Fonts (Cormorant Garamond, Manrope, JetBrains Mono, Noto Sans Arabic). В проде места снимков «Найди отличия» и превью кампаний/уровней занимают реальные иллюстрации игры.

## Files
В этом пакете (`design_handoff_landscape_review/`):

- `Карта кампании - карточки.dc.html` — Журнал уровней (вкл. landscape-кадр).

- `Меню экспедиций.dc.html` — главное меню + paywall + UI-kit (вкл. landscape-кадры).

- `Модалка оценки.dc.html` — оценка игры (вкл. landscape-кадр).

Каждый исходный файл — это «холст» с несколькими экранами (desktop / mobile / landscape / RTL). Для landscape-кадров ориентируйтесь на `data-screen-label`, начинающиеся с `Mobile landscape · …`.
