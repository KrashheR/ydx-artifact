# Sand Meridian — Codex Map Handoff

Этот архив нужен, чтобы Codex мог однозначно сопоставить **13 сюжетных уровней** с **13 точками на пустынной карте**.

## Главный источник правды

Используй `sand-meridian-map-layout.json` как основной data-source.

- `levels[].order` — порядок прохождения;
- `levels[].id` — стабильный ID уровня;
- `levels[].x` / `levels[].y` — координаты точки на карте от 0 до 1;
- `levels[].promptFile` — имя MD-файла сцены из архива с промптами;
- `levels[].landmark` — какой объект на фоне должен находиться под точкой;
- `levels[].storyRole` — сюжетный смысл точки;
- `levels[].visualCues` — элементы, по которым можно сопоставить точку с изображением карты.

## Как размещать точки

Координаты используют систему **normalized-top-left**:

```ts
left = `${point.x * 100}%`;
top = `${point.y * 100}%`;
transform = 'translate(-50%, -50%)';
```

Контейнер карты обязан сохранять `aspect-ratio: 16 / 10`. Фоновое изображение следует отображать через `object-fit: contain`, иначе визуальные landmarks и точки разъедутся.

## Как строить маршрут

1. Отсортировать точки по `order`.
2. Преобразовать normalized-координаты в координаты SVG viewBox `0 0 1600 1000`.
3. Соединить точки одной непрерывной линией.
4. Лучше использовать сглаженный Catmull–Rom/Bezier path.
5. Маршрут не должен пересекаться сам с собой и не должен иметь разрывов.

## Связь с файлами сцен

Папка с промптами уровней должна лежать рядом или импортироваться отдельно. Поле `promptFile` прямо указывает, какой сцене соответствует точка.

Пример:

- `sm-08-oasis-iram`
- точка на карте: `x=0.69`, `y=0.41`
- landmark: зелёный оазис
- файл сцены: `08_oasis-iram.md`

## UX-состояния

Для каждой точки предусмотреть:

- `locked` — закрыта;
- `available` — доступна;
- `completed` — пройдена;
- `current` — текущая;
- `premiumUnlock` — доступна после покупки кампании/пакета, если это используется в проекте.

Рекомендуемая логика доступа: уровень N доступен, когда завершён N-1, кроме уровня 1.

## Что не делать

- Не расставлять точки вручную по пикселям вне конфига.
- Не менять порядок уровней на основании визуальной близости.
- Не встраивать названия уровней в сам фон карты.
- Не использовать `cover` для фонового изображения карты.
- Не генерировать случайные координаты при каждом запуске.
