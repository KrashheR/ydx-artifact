# Production prompt for Claude Design

## Роль

Ты — senior product UI/UX designer браузерных casual-игр и игровых интерфейсов для Yandex Games.

Создай production-ready responsive-дизайн и кликабельный flow publishable vertical slice игры «Найди отличия: Архив аномалий». Это не moodboard и не набор демонстрационных экранов.

## Входные документы

Перед началом обязательно прочитай:

1. `game_concept.json` — единственный источник истины по концепту, scope и правилам.
2. `03_TECHNICAL_STACK_AND_ART_PIPELINE.md` — ограничения реализации и арт-пайплайн.
3. `05_MONETIZATION_I18N_AND_CLOUD_SAVE.md` — обязательные monetization, RU/EN и save states.
4. `06_YANDEX_PLATFORM_RELEASE_AND_MODERATION.md` — platform lifecycle, orientation и catalog states.
5. `07_PRODUCT_ANALYTICS_LIVEOPS_SUPPORT.md` — support, diagnostics и post-release UX.
6. Этот файл.

Не меняй жанр, сеттинг, число уровней, тип меты и модель управления. Не добавляй энергию, жизни, строительство, merge, battle pass, несколько валют или вторую главу.

## Итоговая цель

Подготовь полный UI/UX handoff, который Codex сможет реализовать без догадок.

Дизайн должен:

- объяснять механику за несколько секунд;
- быть читаемым для аудитории 25–60 лет;
- ощущаться как музейный фотоархив и экспедиционная картография;
- быть комфортным на desktop и portrait mobile;
- сохранять игровые фотографии главным визуальным объектом;
- показывать важные состояния, ошибки и fallback;
- не выглядеть как generic mobile puzzle или казино.

## Визуальное направление

Метафора: музейный фотоархив + маршрут северной экспедиции конца 1970-х.

Используй:

- тёплую архивную бумагу;
- паспарту;
- контактные листы;
- негативы;
- металлические зажимы;
- линии маршрута;

- небольшие латунные детали;
- ясную типографику;
- прямоугольные формы с небольшим радиусом.

Опирайся на палитру из `game_concept.json`.

Запрещено:

- неон;

- конфетные градиенты;

Mobile — отдельный UX, а не уменьшенный desktop.

## Критический gameplay UX

### Desktop

- Две фотографии рядом.
- Масштаб и pan синхронизированы.
- Нажатие на отличие работает на любой фотографии.
- Найденное отличие отмечается на обеих фотографиях.
- Игровая область занимает максимум полезного места.
- HUD компактен и не перекрывает сцены.

### Mobile portrait

Основной режим:

- одна фотография занимает почти весь экран;
- кнопка «Сравнить» переключает A/B;
- короткий tap фиксирует другую версию;
- удержание временно показывает другую версию;
- отпускание возвращает предыдущую;
- камера и масштаб не меняются;
- активная версия A/B всегда понятна;
- основные действия доступны большим пальцем.

Дополнительный режим:

- vertical split;
- доступен в настройках;
- не используется по умолчанию.

Не проектируй две миниатюрные фотографии одна над другой как основной mobile UX.

## Карта главы

Карта — линейный маршрут из 12 узлов.

Покажи:

- текущую позицию;
- пройденные уровни;
- количество печатей;
- milestones экспонатов после 3, 6, 9 и 12 уровня;
- daily;
- прогресс главы;
- продолжение маршрута.

Карта должна выглядеть как рабочая карта экспедиции, а не candy-road.

## Результат уровня

Три архивные печати:

- «Восстановлено»;
- «Точный глаз»;
- «Без вмешательства».

Порядок:

1. подтверждение восстановления;
2. появление печатей;
3. награда;
4. продолжение.

Избегай монетных взрывов, casino feedback и полноэкранного конфетти.

## Альбом

Четыре экспоната:

- латунный компас;
- полевая радиостанция;
- засушенный синий цветок;
- разорванная карта.

Для найденного предмета:

- крупная иллюстрация;
- название;
- краткое описание;
- архивный номер;
- дата обнаружения.

Для неизвестного:

- силуэт;
- место получения;
- ясное locked-состояние.

## Daily

Покажи:

- дату;
- сцену;
- мягкий таймер;
- награду;
- streak;
- already-claimed состояние;
- повторное прохождение.

Не превращай daily в агрессивный event hub.

- пять луп;
- эксклюзивная рамка.

## Обязательное дополнение: RU/EN i18n

Production-дизайн поддерживает только:

- Русский;
- English.

Default и fallback — русский.

Обязательно:

- language selector в Settings;
- показать representative English states минимум для Home, Gameplay, Result, Purchase и Error;
- проверить расширение английских строк;
- не использовать текст внутри PNG/WebP;
- не собирать предложения из нескольких визуально раздельных фрагментов;
- предусмотреть dynamic product title/description/price из Yandex catalog;
- показать unknown/missing catalog fallback;
- future locales не проектировать как доступные пользователю.

## Обязательное дополнение: save UX

Yandex cloud — основной progress storage. Локальное зеркало является fallback.

Нужны состояния:

- сохранено в облако;
- сохранение;
- прогресс временно сохранён локально;
- облако недоступно, игра продолжается;
- повторная синхронизация;
- синхронизация восстановлена;
- account selection dialog pause;
- редкий conflict dialog.

Не показывай технические слова SDK, JSON, localStorage или setData.

Игрок не должен блокироваться на бесконечном save spinner. Ошибка облака не запрещает продолжение игры.


## Обязательное дополнение: mobile landscape

Создай отдельный high-fidelity gameplay layout для:

- 640 × 360;
